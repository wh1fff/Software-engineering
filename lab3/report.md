## Ход работы

### 1. Настройка SSH-аутентификации

В ходе выполнения лабораторной работы был сгенерирован SSH-ключ и добавлен в аккаунт GitHub. Ключ был защищён паролем (passphrase). После этого выполнена проверка подключения к GitHub через SSH, которая прошла успешно.

---

### 2. Конфигурация Git

Перед началом работы с репозиторием была выполнена глобальная настройка Git с указанием имени пользователя и адреса электронной почты. Эти данные используются для идентификации автора коммитов. Настройка выполнена через терминал macOS.
```bash
git config --global user.name "александр"
git config --global user.email "aks1de1337@yandex.ru"
git init
```

---

### 3. Инициализация локального репозитория

В рабочей директории был создан новый проект и инициализирован локальный Git-репозиторий. После этого Git начал отслеживать изменения файлов в данной папке. Все действия выполнялись через командную строку без использования графических интерфейсов.

---

### 4. Работа с файлами и коммитами

В процессе выполнения лабораторной работы были созданы и добавлены в репозиторий файлы `hello.py` и `greeting.py`. Все изменения фиксировались с помощью коммитов, что позволило сохранить историю работы с кодом.

**Файл `hello.py`:**
```python
print('Hello')
print('world')
```
Данный файл содержит простую программу на языке Python, выводящую текст в консоль. Он использовался для первичной проверки работы Git и выполнения коммитов.

**Файл `greeting.py`:**
```python
name = input('как вас зовут?')
print(f'приятно познакомиться, {name}')
```
Этот файл реализует простое взаимодействие с пользователем: ввод имени и вывод приветственного сообщения. Корректная работа программы была подтверждена запуском из терминала.

---

### 5. Работа с ветками

В рамках лабораторной работы была создана отдельная ветка `feature/greeting`, предназначенная для разработки новой функциональности. После добавления файла `greeting.py` и создания коммита изменения были успешно слиты с основной веткой `main`.

История коммитов и структура репозитория подтверждают корректность выполнения операции слияния.

---

### 6. Работа с удалённым репозиторием GitHub

После завершения работы с локальным репозиторием был создан удалённый репозиторий на платформе GitHub. Локальный репозиторий был связан с удалённым, после чего ветка `main` и ветка `feature/greeting` были отправлены на GitHub с помощью команды `git push`.

---

### 7. Использование командной строки

Все этапы лабораторной работы выполнялись через терминал в среде macOS. Использование графических инструментов, таких как VS Code, не применялось либо было сведено к минимуму, что позволило лучше освоить работу с Git через консоль.
